Reflogs

git reflog (Reference Log) is a powerful Git command that keeps track of all movements of the HEAD, including checkouts, commits, resets, merges, rebases and reverts. It helps you recover lost commits or switch back to previous states even if they are not part of a branch.

So, it can help us in recovering commits, branches, checkouts etc.

Git Stash

It is a powerful command in Git that allows you to temporarily save changes in your working directory that you aren't ready to commit yet. It stores your modified (and optionally staged) changes in a "stash" and reverts your working directory to the last committed state. This is especially useful if you need to switch branches or work on something else without losing your uncommitted changes.

    commands

  1. git stash
    2. This saves your changes (both staged and unstaged) and resets your working directory to match the last commit.
  2. git stash list
    3. Lists all stashes indexed.
  3. git stash apply
    4. To apply changes of most recent stash but keeps in stash list.
  4. git stash apply stash@{index}
    5. To apply a specific stash.
  5. git stash drop stash@{index}
  6. git stash pop
    7. This applies the recent stash and removes it from stash list.
  7. git stash push -k
    8. Stashes only unstaged changes.
  8. git stash push -m "Your message here"
    9. Adds a descriptive message to your stash.
  9. git stash clear
    10. To remove all stashes
